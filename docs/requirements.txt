1. PROJECT OVERVIEW

1.1 Business Model
The system acts as an intermediary platform connecting customers with a restaurant. The client company:
‚Ä¢	Receives orders from customers through website and mobile application
‚Ä¢	Manually calls the restaurant to place orders
‚Ä¢	Provides delivery service through registered riders
‚Ä¢	Collects payment from customers (including service fee and delivery charges)
‚Ä¢	Pays the restaurant for food items separately

1.2 Platform Description
Website
Customer facing web platform displaying restaurant menu, allowing food ordering, payment processing, and order tracking.

Mobile Application
Dual purpose mobile app (Android & iOS) with role based interfaces:
‚Ä¢	Customer Interface: Browse menu, place orders, track delivery, manage profile
‚Ä¢	Rider Interface: Receive order notifications, accept deliveries, update order status, view earnings

Admin Panel
Web based management system for order processing, menu management, rider management, pricing configuration, and reporting.

1.3 Initial Scope
Phase 1: Two restaurant branches implementation
Future: Expandable to multiple branches across Sri Lanka

‚Ä¢	Order Processing: After admin calls restaurant and confirms order

‚Ä¢	Delivering: During transit to customer location

‚Ä¢	This ensures secure handover and prevents delivery fraud

‚Ä¢	New order alert with sound/visual notification on dashboard

‚Ä¢	Manually call restaurant and create order

‚Ä¢	View all order details and customer information
‚Ä¢	Order history and search functionality

Pricing Structure
‚Ä¢	Restaurant Food Price: Base price set by restaurant

‚Ä¢	Delivery Fee: Distance-based calculation (price per km configurable in admin panel)
‚Ä¢	Rider Tip: Optional, added by customer at checkout
‚Ä¢	Total = Food Price + Service Fee + Delivery Fee + Tip

Admin Pricing Configuration
‚Ä¢	Set and update service fee percentage
‚Ä¢	Set and update delivery charge per kilometer
‚Ä¢	Manage branch-wise delivery radius (see Branch & Location Management section)

4.8 Branch & Location Management
Branch Setup (Admin Panel)
‚Ä¢	Add new restaurant branches with:
-	Branch name 
-	Branch address 
-	GPS coordinates (latitude/longitude) 
-	Delivery radius in kilometers (each branch has its own radius) 
-	Branch contact number
-	Operating hours
‚Ä¢	Edit or deactivate existing branches
‚Ä¢	Manage menu availability per branch
‚Ä¢	Currently configured for 2 branches

User Location Detection
‚Ä¢	When user opens the app/website, system automatically detects user's GPS location
‚Ä¢	System calculates distance from user to ALL active branches
‚Ä¢	Identifies the nearest branch to user's location
‚Ä¢	Checks if user is within that branch's delivery radius

Branch Selection Logic
‚Ä¢	Step 1: Get user's current GPS location
‚Ä¢	Step 2: Calculate distance to all branches
‚Ä¢	Step 3: Find nearest branch
‚Ä¢	Step 4: Check if user is within that branch's delivery radius
-	If YES: Display that branch's menu and allow ordering
-	If NO: Check next nearest branch
-	If NO branch covers user's location: Display "Delivery not available in your area




Per-Branch Delivery Radius
‚Ä¢	Each branch has its own configurable delivery radius (in KM) ¬∑ 
‚Ä¢	Admin can set different radius for each branch based on location needs 
‚Ä¢	This allows expansion to any location in Sri Lanka¬∑ 
‚Ä¢	Urban branches can have smaller radius, rural branches can have larger radius

Order Processing by Branch
‚Ä¢	Orders are automatically assigned to the branch serving that customer's location ¬∑ 
‚Ä¢	Admin dashboard shows orders filtered by branch ¬∑ 
‚Ä¢	Each branch manages its own orders independently ¬∑ 
‚Ä¢	Riders are assigned to orders from their designated branch
